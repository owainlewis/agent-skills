# Title Reference

The title process lives in `produce.md` (youtube skill). This file is reference material — patterns and examples to draw from when generating titles.

## The Good Title Checklist

Every title must pass all four:

- **Short?** — Under 10 words. Ideally under 8.
- **Bold?** — Makes a claim or takes a position.
- **Original?** — Not five other videos with the same title on YouTube.
- **Intriguing?** — Creates a question in the viewer's mind.

## Word Swap Test

Before finalising, read every word and ask: is there a simpler one?

| Replace | With |
|---------|------|
| utilise | use |
| implement | build |
| comprehensive | full |
| autonomous | cut it |
| fundamentals | basics |
| optimise | fix / improve |
| architecture | setup / structure |

If the audience knows the acronym, use it: "retrieval-augmented generation" → "RAG"

## Patterns Worth Stealing

These aren't rules — they're starting points for brainstorming.

**Contrarian:** "Why [common belief] is [wrong]"
**Process reveal:** "How I [specific thing]"
**Time-boxed:** "[Result] in [time]"
**Private question:** "[Question the viewer is already asking themselves]"
**Compression:** "[Big effort] in [small package]"
**Clearly Explained:** "[Trending concept], Clearly Explained"
**Personal setup:** "My [specific setup/workflow/system]"
**Bold claim:** "[Tool] is all you need"
**Branded system:** "[Named System]: The Missing [X]"
**Transformation:** "Replacing [old] with [new]"

### Performance Tier

Personal and possessive titles ("My...", "How I...") consistently outperform generic advice titles ("Stop doing X", "The Complete Guide"). When generating titles, weight personal framing higher. Evidence: across multiple dev/AI channels, "My..." and "How I..." patterns appear disproportionately in top-performing videos relative to how often they're used.
